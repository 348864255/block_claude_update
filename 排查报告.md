# Claude Store 应用自动更新排查报告

**排查日期**：2026-06-23  
**系统**：Windows 11 Home China (10.0.26200)  
**目标版本**：Claude `1.12603.1.0`  
**被自动更新到**：Claude `1.14271.0.0`（今天 13:12 被写入）

---

## 一、现象

`C:\Program Files\WindowsApps\` 下出现两个版本目录：
```
Claude_1.12603.1.0_x64__pzs8sxrjxfjjc   创建时间：2026/6/23 9:11
Claude_1.14271.0.0_x64__pzs8sxrjxfjjc   创建时间：2026/6/23 13:12  ← 被自动下载
```

已尝试过：
- ✅ 防火墙封锁 Store 客户端进程 → **无效**
- ✅ 注册表 `AutoDownload=2` → **无效**

---

## 二、排查过程

### 2.1 排除 Windows Update

```powershell
# 生成 WU 日志并搜索 Claude
Get-WindowsUpdateLog
Select-String -Path $env:TEMP\WindowsUpdate.log -Pattern 'Claude|pzs8sxrjxfjjc'
# 结果：无任何匹配 → 更新不走 Windows Update
```

**结论**：Windows Update 日志中无 Claude 相关记录，排除此通道。

### 2.2 发现 Delivery Optimization 是下载执行者

```powershell
Get-DeliveryOptimizationStatus | Format-List
```

关键输出：
```
PredefinedCallerApplication : WU Client Download
SourceURL                   : http://1d.tlu.dl.delivery.mp.microsoft.com/filestreamingservice/...
Status                      : Caching
TotalBytesDownloaded        : 1221644925  (≈1.16GB)
DownloadDuration            : 00:32:23
```

**结论**：Delivery Optimization 从微软 CDN 下载了约 1.2GB 内容，耗时 32 分钟。

### 2.3 发现 ContentDeliveryManager 是决策者

```powershell
Get-ItemProperty 'HKCU:\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager'
```

关键发现：
```
SilentInstalledAppsEnabled : 1  ← 允许后台静默安装 Store 应用更新
ContentDeliveryAllowed     : 1  ← 允许微软推送内容（含应用更新）
```

**结论**：这两个开关是**根因**。ContentDeliveryManager 决定"静默安装"，然后调用 DO 执行下载。

### 2.4 发现 InstallService 计划任务是触发器

```powershell
Get-ScheduledTask -TaskPath '\Microsoft\Windows\InstallService\*' | Select TaskName, State
```

关键发现：
```
ScanForUpdates       Ready  ← 每日定时触发 Store 应用更新扫描
ScanForUpdatesAsUser Ready  ← 以用户身份扫描
RestoreDevice        Ready  ← 设备恢复任务
```

**结论**：`ScanForUpdates` 每日触发，通知系统检查 Store 应用更新。

### 2.5 AutoDownload=2 为什么没用？

```powershell
# 已设置的策略
reg query "HKLM\SOFTWARE\Policies\Microsoft\WindowsStore" /v AutoDownload
# AutoDownload    REG_DWORD    0x2
```

**原因**：`AutoDownload=2` 只控制 Store **客户端**在非 Wi-Fi 下的行为。ContentDeliveryManager 是 Windows 系统级通道，**独立于 Store 客户端，不受此策略控制**。

## 三、更新链路还原

```
InstallService\ScanForUpdates（计划任务，每日触发）
  → 通知系统检查 Store 应用更新
    → ContentDeliveryManager 判断"静默安装"
      → 调用 Delivery Optimization 下载安装包
        → 从 dl.delivery.mp.microsoft.com (微软 CDN) 下载
          → 写入 WindowsApps\Claude_1.14271.0.0_x64__...
```

**关键洞察**：防火墙封的是 Store 客户端进程（WindowsStore.exe），但这套链路走的是 **DO 服务 + 微软 CDN**，完全绕过了 Store 客户端。

---

## 四、修复方案（已执行）

### 步骤 1：关闭 ContentDeliveryManager（根因）
```cmd
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v SilentInstalledAppsEnabled /t REG_DWORD /d 0 /f
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v ContentDeliveryAllowed /t REG_DWORD /d 0 /f
```

### 步骤 2：禁用 Delivery Optimization 服务
```cmd
sc stop DoSvc
reg add "HKLM\SYSTEM\CurrentControlSet\Services\DoSvc" /v Start /t REG_DWORD /d 4 /f
```
> ⚠️ `sc config DoSvc start= disabled` 可能因权限被拒，用注册表 `Start=4` 绕过。

### 步骤 3：禁用 InstallService 计划任务
```cmd
schtasks /Change /TN "\Microsoft\Windows\InstallService\ScanForUpdates" /Disable
schtasks /Change /TN "\Microsoft\Windows\InstallService\ScanForUpdatesAsUser" /Disable
schtasks /Change /TN "\Microsoft\Windows\InstallService\RestoreDevice" /Disable
```

### 步骤 4：禁用 Delivery Optimization 策略
```cmd
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\DeliveryOptimization" /v DODownloadMode /t REG_DWORD /d 0 /f
```

### 步骤 5：重启电脑

## 五、验证方法

重启后执行：
```cmd
:: 1. 确认 CDM 已关闭
reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v SilentInstalledAppsEnabled
reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v ContentDeliveryAllowed

:: 2. 确认 DoSvc 已禁用
sc qc DoSvc
:: 应显示 START_TYPE : 4  DISABLED

:: 3. 确认计划任务已禁用
schtasks /Query /TN "\Microsoft\Windows\InstallService\ScanForUpdates" /FO LIST | findstr Status

:: 4. 24小时后检查是否还有新版本目录
dir "C:\Program Files\WindowsApps\Claude*"
```

## 六、恢复方法

以管理员身份运行 `restore_claude_update.bat`（见桌面），或手动：
```cmd
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v SilentInstalledAppsEnabled /t REG_DWORD /d 1 /f
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v ContentDeliveryAllowed /t REG_DWORD /d 1 /f
reg add "HKLM\SYSTEM\CurrentControlSet\Services\DoSvc" /v Start /t REG_DWORD /d 2 /f
sc start DoSvc
schtasks /Change /TN "\Microsoft\Windows\InstallService\ScanForUpdates" /Enable
schtasks /Change /TN "\Microsoft\Windows\InstallService\ScanForUpdatesAsUser" /Enable
schtasks /Change /TN "\Microsoft\Windows\InstallService\RestoreDevice" /Enable
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\DeliveryOptimization" /v DODownloadMode /f
```

## 七、经验总结

| 教训 | 说明 |
|------|------|
| **不要只封 Store 客户端** | Windows Store 应用更新走的是 Delivery Optimization + CDN，不是 Store 客户端进程 |
| **AutoDownload=2 不够** | 只影响 Store 客户端行为，ContentDeliveryManager 是独立通道 |
| **ContentDeliveryManager 是根因** | `SilentInstalledAppsEnabled` 控制后台静默安装，必须关闭 |
| **DoSvc 用注册表禁用** | `sc config` 可能权限不够，直接改注册表 `Start=4` 更可靠 |
| **三道防线都要做** | CDM + DoSvc 服务 + InstallService 计划任务，缺一不可 |
| **按流量计费是双保险** | 设置网络为"按流量计费"可额外阻止 Windows 后台下载 |
